# 考公App 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品定位
一款由**个人开发者**打造的、以**AI智能学习**为核心的公务员考试备考工具，覆盖从零基础入门到冲刺提分的全阶段考生需求。通过AI技术实现个性化学习路径规划、智能刷题推荐、错题深度分析等核心能力，帮助考生高效备考。

> **项目性质**：个人开源/免费项目，不以商业盈利为目的，专注于为考公群体提供优质的AI辅助学习体验。所有核心功能免费开放，未来如接入第三方大模型API产生成本，仅考虑接受用户自愿打赏/赞助以覆盖基础服务器和API调用费用。

### 1.2 产品愿景
让每一位考公人拥有专属的AI学习助手，告别"题海战术"，实现精准高效备考。

### 1.3 产品名称
**智考**

### 1.4 产品形态
- **一期（MVP）**：Web端（Flutter Web），快速验证产品核心功能和用户体验
- **二期**：迁移至移动端 App（iOS + Android，Flutter 原生）
- **三期**：微信小程序（轻量化入口）

## 四、核心功能详细设计

### 4.1 AI学习中心（核心模块）

#### 4.1.1 AI学习路径规划

**功能描述**：根据用户的基础水平、备考时间、目标分数，AI自动生成个性化学习计划。

**详细流程**：
1. **入学测评**（首次使用）
   - 用户选择考试类型（国考/省考）+ 目标岗位
   - 完成一套基础能力诊断题（约30分钟，覆盖行测各模块）
   - 填写备考时间（每天可用学习时长、距离考试天数）
   - 设定目标分数

2. **AI生成学习计划**
   - 基于诊断结果，分析各科目/模块的掌握程度（0-100%）
   - 生成可视化能力雷达图
   - 输出个性化学习路径：
     - 分阶段计划：基础阶段 → 强化阶段 → 冲刺阶段
     - 每日任务：今天需要完成的具体学习内容
     - 预估目标达成率

3. **动态调整**
   - 根据每日学习完成情况自动调整后续计划
   - 某模块掌握度达标后自动减少该模块练习量
   - 学习进度落后时提醒并压缩计划

**AI技术点**：
- 基于知识图谱的学习路径推荐算法
- 自适应学习模型（类似IRT项目反应理论）
- 时间规划优化算法

**输出示例**：
```
你的专属学习计划（距离国考还有 180 天）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
当前综合水平：52/100
目标分数：135分（行测70 + 申论65）
预计达成概率：78%

今日任务：
□ 行测 · 言语理解 · 片段阅读精练 20题（约30min）
□ 申论 · 归纳概括 · 基础方法论学习（约45min）
□ 时政 · 每日热点速记 3条（约10min）
□ 复习昨日错题 5题（约15min）

本周重点：言语理解专项突破
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 4.1.2 智能刷题

**功能描述**：基于AI算法，为每个用户推荐最适合当前水平的练习题，实现"千人千面"的刷题体验。

**详细设计**：

1. **刷题模式**
   | 模式 | 说明 | 适用场景 |
   |------|------|----------|
   | AI推荐模式 | 系统自动推荐最适合的题目 | 日常练习 |
   | 专项练习 | 用户自选科目/考点 | 查漏补缺 |
   | 每日挑战 | 每天一组精选题，完成后有奖励 | 碎片化学习 |
   | 模拟考试 | 限时仿真考试 | 阶段测评 |
   | 错题重做 | 针对之前做错的题再次练习 | 巩固提升 |
   | 收藏练习 | 练习用户收藏的题目 | 回顾重点 |

2. **AI推荐算法逻辑**
   - **难度匹配**：根据用户当前水平推荐略高于当前能力的题目（最近发展区理论）
   - **知识点覆盖**：确保所有薄弱知识点都被充分练习
   - **间隔重复**：根据艾宾浩斯遗忘曲线安排已学知识点的复习
   - **新鲜度**：避免短期内出现重复题目
   - **实时调整**：答题过程中根据正确率动态调整后续题目难度

3. **两种答题模式**

   **模式一：逐题即时反馈模式（练习模式）**
   - 每做完一题，立即显示对错和答案解析
   - 适合日常刷题、碎片化练习
   - 做错后可立即查看解析并标注错因（见4.1.3错因标注）
   - 即时反馈有助于加深记忆、快速纠正

   ```
   ┌─────────────────────────────────┐
   │  第 3/20 题          01:32       │
   ├─────────────────────────────────┤
   │                                 │
   │  题干内容........................ │
   │                                 │
   │  A. 选项一                       │
   │  B. 选项二  ✓ 你的选择           │
   │  C. 选项三                       │
   │  D. 选项四                       │
   │                                 │
   ├─────── 回答正确 ✓ ──────────────┤
   │  📖 解析：                       │
   │  本题考查xxx知识点...            │
   │                                 │
   │  ┌──────┐  ┌──────┐  ┌──────┐  │
   │  │下一题│  │ 收藏 │  │举一反三│  │
   │  └──────┘  └──────┘  └──────┘  │
   └─────────────────────────────────┘
   ```

   **模式二：整套做完再出结果模式（试卷模式）**
   - 做完一套完整试题后才统一展示成绩和每道题的答案解析
   - 适合模拟考试、历年真题练习、自我测评
   - 支持答题卡，可跳题、标记疑难题、回顾已答题目
   - 提交试卷后展示成绩单 + 逐题解析 + 错因标注入口

   ```
   ┌─────────────────────────────────┐
   │  2025年国考行测真题（副省级）      │
   │  限时：120分钟       剩余：87:32  │

   └─────────────────────────────────┘
   ```

   **两种模式对比**：
   | 维度 | 逐题即时反馈 | 整套做完再出结果 |
   |------|-------------|-----------------|
   | 答题节奏 | 做一题看一题 | 连续做完所有题 |
   | 反馈时机 | 每题即时 | 提交后统一 |
   | 错因标注 | 做错后立即标注 | 查看解析时逐一标注 |
   | 适用场景 | 日常练习、AI推荐、专项刷题 | 模拟考试、真题套卷、自我测评 |
   | 时间压力 | 无（不限时或宽松限时） | 有（严格限时，仿真考试） |
   | 用户选择 | 默认模式，进入练习即此模式 | 需选择"试卷模式"或"模拟考试" |

4. **其他答题辅助功能**
   - 做题计时（单题用时统计，帮助发现耗时过长的题型）
   - 题目收藏（做题过程中随时收藏重点题目）
   - 题目标记（试卷模式下标记疑难题，方便回来检查）
   - 做题笔记（在题目旁添加个人笔记）

5. **即时反馈与解析**
   - 显示正确/错误状态
   - **AI生成解析**：
     - 正确答案解析（为什么选这个）
     - 错误选项分析（每个错误选项的迷惑点在哪）
     - 考点归纳（这道题考查了什么知识点）
     - 举一反三（推荐3道相似题供巩固）
     - 做题技巧（这类题的快速解题方法）

#### 4.1.3 AI错题分析

**功能描述**：不只是记录错题，而是通过AI深度分析错因，给出针对性提升建议。

**详细设计**：

1. **错题自动归档**
   - 练习/考试中的错题自动进入错题本
   - 支持手动添加错题（拍照录入）
   - 自动分类：按科目 → 模块 → 考点 → 错因

2. **用户错因标注**（核心交互功能）

   用户做错题目后，系统会引导用户进行错因标注。这一功能帮助用户主动反思错误原因，同时为AI个性化推荐提供更精准的数据。

   **标注入口**：
   - 逐题模式：做错后立即弹出标注弹窗
   - 试卷模式：查看解析时，每道错题旁有「标注错因」按钮
   - 错题本：浏览历史错题时可随时补充/修改错因标注

   **标注方式 = 预设选项 + 自由输入**：

   ```
   ┌─────────────────────────────────┐
   │  ❌ 回答错误                     │
   │                                 │
   │  正确答案：C  你的答案：B         │
   │                                 │
   │  📖 解析：本题考查xxx...         │
   │                                 │
   ├─────── 为什么做错了？ ──────────┤
   │                                 │
   │  请选择错因（可多选）：           │
   │                                 │
   │  ┌─────────────────────────┐    │
   │  │ ○ 知识盲区 — 不会做      │    │
   │  │ ● 概念混淆 — 记混了      │    │
   │  │ ○ 审题失误 — 没看清题目  │    │
   │  │ ○ 计算错误 — 算错了      │    │
   │  │ ○ 粗心大意 — 看错/写错   │    │
   │  │ ○ 方法不对 — 思路错误    │    │
   │  │ ○ 时间不够 — 来不及做    │    │
   │  │ ○ 猜的 — 完全蒙的       │    │
   │  └─────────────────────────┘    │
   │                                 │
   │  补充说明（选填）：               │
   │  ┌─────────────────────────┐    │
   │  │ 记录你的思考过程...       │    │
   │  │                         │    │
   │  └─────────────────────────┘    │
   │                                 │
   │  ┌──────────┐  ┌──────────┐    │
   │  │   跳过   │  │ 提交标注  │    │
   │  └──────────┘  └──────────┘    │
   └─────────────────────────────────┘
   ```

   **预设错因选项详细说明**：
   | 错因选项 | 说明 | 图标 |
   |----------|------|------|
   | 知识盲区 | 相关知识点完全不会，无从下手 | 📕 |
   | 概念混淆 | 相似概念/公式记混了 | 🔀 |
   | 审题失误 | 没看清题目要求或关键条件 | 👁️ |
   | 计算错误 | 思路对但中间计算过程出错 | 🔢 |
   | 粗心大意 | 看错选项、填错答案等低级失误 | 💨 |
   | 方法不对 | 有思路但解题方法不正确/不最优 | 🧭 |
   | 时间不够 | 来不及思考或做不完 | ⏱️ |
   | 猜的 | 完全蒙的答案 | 🎲 |

   **自由输入框**：
   - 最多200字，支持文本输入
   - 用于记录用户自己的思考过程、疑惑点
   - 示例："这道数量关系题我用了方程法，但计算太复杂了，应该用比例法更快"
   - AI会分析用户填写的内容，辅助判断真实错因

   **标注行为设计**：
   - 错因标注为**可选操作**，不强制要求（避免打断学习节奏）
   - 用户可以多选错因（一道题可能有多个原因）
   - 支持事后补充或修改错因标注
   - 标注数据同步影响AI推荐策略和错因分析报告

3. **AI辅助错因分析**

   除了用户自主标注，AI也会基于答题数据进行自动错因分析：

   | 分析维度 | AI判断依据 | 示例 |
   |----------|-----------|------|
   | 答题耗时 vs 正确率 | 耗时长且做错 → 可能是知识盲区 | 一道题花了3分钟还选错 |
   | 同类题目表现 | 同考点反复出错 → 概念未掌握 | 言语理解中的逻辑填空连续错 |
   | 答案选择倾向 | 系统性选错某个干扰项 → 概念混淆 | 总是混淆充分条件和必要条件 |
   | 时间分布 | 后半部分正确率骤降 → 时间不够 | 试卷后半部分大量未答或乱选 |
   | 修改行为 | 改对→改错 → 粗心/犹豫 | 原本选对了又改错 |

   **最终错因 = 用户标注（主）+ AI分析（辅）**：
   - 用户标注了错因 → 以用户标注为准
   - 用户未标注 → 显示AI分析的推测错因（标注为"AI推测"）
   - 两者不一致 → 同时展示，供用户确认

4. **错题智能复习**
   - 基于间隔重复算法安排错题复习
   - 错题复习周期：1天 → 3天 → 7天 → 15天 → 30天
   - 同一道题连续答对3次则标记为"已掌握"
   - 定期生成错题专项测试卷

4. **错题报告**
   - 错题数量趋势图（周/月）
   - 错因分布饼图
   - 薄弱知识点排名
   - 同类错题归类展示

### 4.3 数字敏感度训练

#### 4.3.1 功能定位

资料分析和数量关系是行测中得分率波动最大的两个模块，核心瓶颈往往不在"会不会做"，而在**对数字的敏感度不够**——算得慢、找数慢、比大小慢、估算不准。本模块通过短时高频的专项训练，帮助用户从底层提升数字处理能力，为资料分析和数量关系打好基本功。

> 设计理念：每天5-10分钟，像练钢琴的"音阶练习"一样，通过反复训练建立数字直觉。

#### 4.3.3 基础速算练习

**目标**：提升加减乘除的心算速度和准确率，这是资料分析的基础功。

**训练模式**：限时连续作答，每轮20题，答完自动计分。

| 练习类型 | 说明 | 示例 | 难度分级 |
|----------|------|------|----------|
| 加减法速算 | 两位数/三位数的加减运算 | 347 + 258 = ? | 两数相加 → 多数连加 → 带小数 |
| 乘法速算 | 重点训练两位数乘一位数、特殊乘法技巧 | 25 × 36 = ? | 一位数乘法 → 特殊数乘法(×25/×125) → 任意两位数 |
| 除法速算 | 整除判断、估算商的范围 | 847 ÷ 6 ≈ ? | 整除 → 估算到整数 → 估算到小数点后一位 |
| 混合速算 | 综合四则运算 | 3.6 × 2.5 + 1.8 = ? | 两步运算 → 三步运算 |

**交互设计**：
```
┌─────────────────────────────────┐
│  加减法速算  第 12/20 题        │
│  ⏱ 已用时 00:38   正确 10/11    │
├─────────────────────────────────┤
│                                 │
│          437 + 286 = ?          │
│                                 │
│     ┌─────────────────────┐    │
│     │ 723                 │    │
│     └─────────────────────┘    │
│                                 │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐     │
│  │ 1 │ │ 2 │ │ 3 │ │ ← │     │
│  ├───┤ ├───┤ ├───┤ ├───┤     │
│  │ 4 │ │ 5 │ │ 6 │ │ → │     │
│  ├───┤ ├───┤ ├───┤ ├───┤     │
│  │ 7 │ │ 8 │ │ 9 │ │ ✓ │     │
│  ├───┤ ├───┤ ├───┤ ├───┤     │
│  │ . │ │ 0 │ │清除│ │提交│     │
│  └───┘ └───┘ └───┘ └───┘     │
└─────────────────────────────────┘
```

- 数字键盘输入，减少键盘操作时间
- 每题即时反馈对错，答错显示正确答案
- 提交后展示成绩：总用时、正确率、平均每题用时、与上次对比

**速算技巧提示**（每轮练习后推送一条）：
- 尾数法：只看尾数快速判断答案
- 凑整法：把数凑成整十整百再计算
- 特殊值法：×25等于÷4×100、×125等于÷8×1000

#### 4.3.6 读表/读图训练

**目标**：训练快速理解表格结构和图表含义的能力，缩短资料分析中"读题→理解材料"的时间。

**1. 表格速读**

给出一个数据表格，限时回答结构理解类问题：
- "表格的行标题代表什么？"
- "哪一列数据代表同比增速？"
- "表格中最大的值出现在哪一行？"
- "合计行与各分项之和是否一致？"

**2. 图表速读**

给出一个统计图表，限时回答：
- "这个柱状图的横轴/纵轴分别代表什么？"
- "折线图中有几条线，分别代表什么？"
- "饼图中占比最大的是哪一项？"
- "图表的单位和图例分别是什么？"

**3. 图表陷阱识别**

训练识别常见图表陷阱：
- 坐标轴不从0开始（夸大差异）
- 饼图各部分之和不等于100%
- 双纵轴刻度不同导致的视觉误导
- 折线图时间间隔不一致

与徽章不同，里程碑是**数值型的进度记录**，展示用户的学习轨迹：

```
┌─────────────────────────────────┐
│  学习里程碑                     │
├─────────────────────────────────┤
│                                 │
│  📅 累计学习 47 天              │
│  ├── 下一个：60天（备考习惯巩固）│
│                                 │
│  📝 累计做题 1,847 道           │
│  ├── 下一个：2,000道            │
│                                 │
│  ✅ 综合正确率 76%              │
│  ├── 下一个：80%（稳步提升中）  │
│                                 │
│  ⏱ 累计学习 89.5 小时          │
│  ├── 下一个：100小时            │
│                                 │
│  🔥 当前连续学习 12 天          │
│  ├── 最高纪录：23天             │
│  └── 下一个：30天（月度标兵）   │
│                                 │
├─────────────────────────────────┤
│  ⏳ 距离国考还有 143 天         │
│  当前进度处于 前 35%            │
└─────────────────────────────────┘
```

#### 4.8.5 排行榜系统

**设计原则**：排行榜以**激励为主**，避免让排名靠后的用户产生挫败感。

**1. 排行榜类型**

| 排行榜 | 维度 | 说明 |
|--------|------|------|
| 今日学习榜 | 当日做题量 | 每日重置，所有人都有机会上榜 |
| 本周勤奋榜 | 本周学习时长 | 7天周期 |
| 连续打卡榜 | 当前连续学习天数 | 长期维度，鼓励坚持 |
| 速算挑战榜 | 当日速算最佳成绩 | 数字敏感度专属排行 |
| 进步榜 | 本周正确率提升幅度 | 只看进步不看绝对值，对新手友好 |

**2. 排行展示方式**
- 不展示绝对排名，展示**百分位**
- "你今天的学习量超过了 82% 的考生"
- 比排名数字更友好，不会打击排名靠后的用户
- 前三名可以展示昵称（用户可选择匿名）

**3. 隐私保护**
- 默认匿名参与排行
- 用户可在设置中选择是否公开昵称
- 不展示具体分数，只展示相对位置

#### 4.8.6 激励通知设计

徽章和里程碑的触发通知，用温和有趣的语言：

| 触发事件 | 通知文案示例 |
|----------|-------------|
| 获得"一周达人" | 连续学习7天！你已经度过了最难的第一周，继续加油 |
| 连续答对10题 | 10连对！手感火热，趁热打铁再来几道？ |
| 做题达到1000 | 千题斩达成！你已经是千里挑一的考生了 |
| 周正确率提升 | 本周正确率提升了8%，你的努力看得见 |
| 连续3天未学习 | 好几天没见了，今天的题在等你哦 |
| 即将解锁徽章 | 再做12道题就能获得"言语大师"徽章了 |

#### 4.8.7 与其他模块的联动

- **AI学习路径**：学习计划中标注"完成今日任务可获得xxx徽章进度"
- **数字敏感度训练**：速算/找数成绩直接关联数感类徽章
- **错题本**："错题清零"徽章鼓励用户持续复习错题
- **学习报告**：周报/月报中展示本期获得的徽章和里程碑进展
- **首页**：最新获得的徽章在首页短暂展示（成就感外化）

#### 4.8.8 数据存储

成就系统数据结构简单，只需记录：
- 用户已获得的徽章列表（徽章ID + 获得时间）
- 里程碑当前值（各维度数值）
- 排行榜数据（每日/每周聚合）

开发量小，与现有用户行为数据复用，不增加额外埋点负担。

### 4.9 题目评论与考生互助

#### 4.9.1 功能定位

AI解析虽然强大，但有时候**另一个考生的真实备考经验**比标准答案更有启发性。本模块允许用户对题目发表评论、分享做题心得、提供自己的解题思路，形成"考生互助"的知识沉淀。

> 设计理念：AI + 人际双轨学习，让每道题的解析不只是标准答案，而是一个"多元视角的讨论场"。

#### 4.9.2 题目评论系统

**评论入口**：每道题的详情/解析页面底部

```
┌─────────────────────────────────┐
│  📖 题目解析                     │
│  ...（解析内容）                  │
├─────── 考生讨论 (23) ───────────┤
│                                 │
│  🧑 小李  ·  2小时前             │
│  这题有个更快的方法：看到"至少"  │
│  就想到最不利原则，直接2×4+1=9  │
│  👍 12  💬 3                    │
│    ├── 🧑 阿花：对！秒杀！      │
│    └── 🧑 大白：原来如此，记了   │
│                                 │
│  🧑 数量关系渣  ·  昨天          │
│  我的错因：把"至少"看成"至多"  │
│  了，审题一定要仔细！大家注意   │
│  这个坑。                       │
│  👍 8  💬 1                     │
│                                 │
│  🧑 考公第三年  ·  3天前         │
│  这道题2023年国考第47题几乎一样 │
│  就是换了数字，同考点的题还有   │
│  2022年第39题，建议一起做。     │
│  👍 21  💬 5                    │
│                                 │
├─────────────────────────────────┤
│  ┌─────────────────────────┐   │
│  │ 分享你的做题心得...       │   │
│  └─────────────────────────┘   │
│  ┌──────────────────────────┐  │
│  │ 发表评论                  │  │
│  └──────────────────────────┘  │
└─────────────────────────────────┘
```

**评论功能**：
- 发表评论（文字，最多500字）
- 回复他人评论（支持二级回复）
- 点赞（不设踩，避免负面氛围）
- 评论排序：默认按"最热"（点赞数），可切换为"最新"
- AI自动审核：接入LLM判断评论是否合规（无广告、无攻击、与题目相关）

**评论类型标签**（用户发表时可选择标签）：
| 标签 | 含义 | 示例 |
|------|------|------|
| 快捷方法 | 比解析更快的解题技巧 | "秒杀法：直接看尾数" |
| 易错提醒 | 提醒常见错误和陷阱 | "注意题目说的是'不正确'的" |
| 举一反三 | 关联其他类似题目 | "这题和2022年国考第39题类似" |
| 错因分享 | 分享自己做错的原因 | "把'至少'看成'至多'了" |
| 知识补充 | 补充相关知识点 | "考的是三段论的有效式" |

#### 4.9.3 考生互助系统

在题目评论的基础上，进一步实现"用户之间的经验共享"：

**1. 错题笔记推荐**
- 用户A做错一道题后写了详细的心得笔记（错因标注中的"补充说明"字段）
- 系统将这条笔记脱敏后推荐给做错同一道题的用户B
- 展示形式：在用户B查看该题解析时，额外显示"其他考生的经验"

```
┌─────────────────────────────────┐
│  ❌ 你做错了这道题               │
├─────────────────────────────────┤
│  📖 AI解析                      │
│  本题考查最不利原则...           │
│                                 │
├────── 考过的同学说 ─────────────┤
│                                 │
│  💡 "看到'至少保证'就是最不利   │
│     原则的信号词，先按最不利的   │
│     情况取，再加1就行了。"      │
│                          — 匿名 │
│                                 │
│  💡 "这题我做错是因为没考虑到   │
│     极端情况，建议遇到'至少'    │
│     先想最坏情况。"             │
│                          — 匿名 │
│                                 │
│  ┌──────────────────────────┐  │
│  │ 也想分享你的经验？         │  │
│  └──────────────────────────┘  │
└─────────────────────────────────┘
```

**2. 互助贡献激励**
- 用户的评论被点赞 → 获得互助贡献分
- 评论被评为"有帮助"的次数越多 → 解锁"热心助考"徽章
- 个人中心展示"帮助了X位考生"

**3. 内容审核机制**

| 审核层 | 方式 | 说明 |
|--------|------|------|
| 第一层 | AI自动审核 | LLM判断评论是否合规、与题目相关 |
| 第二层 | 用户举报 | 其他用户可举报不当评论 |
| 第三层 | 开发者审核 | 举报内容由开发者人工审核（初期） |

- 优先依赖AI审核，降低人工成本
- 明确的社区规范：禁止广告、人身攻击、无关内容
- 违规用户警告 → 限制评论 → 封禁

#### 4.9.4 数据模型

```
题目评论表 (question_comment)
├── id
├── question_id（关联题目）
├── user_id（评论者，对外展示可选匿名）
├── parent_id（回复的评论ID，null为顶级评论）
├── tag（评论类型标签）
├── content（评论内容，≤500字）
├── like_count（点赞数）
├── status（状态：正常/审核中/已隐藏）
├── created_at
└── updated_at
```

#### 4.9.5 与其他模块的联动

- **错题本**：错因标注中的"补充说明"可一键转为题目评论
- **AI学习助手**：用户问AI的问题如果被多人重复问到，系统提示"这是高频疑问，查看考生讨论"
- **学习成就**：评论被点赞达一定次数可解锁"热心助考"徽章
- **首页推荐**：热门讨论（点赞多的评论）可在首页展示

## 七、运营模式设计

> 本产品定位为个人开发者的非商业项目，所有核心功能免费开放。

### 7.1 免费开放策略
- 所有核心功能完全免费（题库、AI分析、错题本、学习报告等）
- 不设功能限制、不做每日使用次数限制
- 不设会员等级，所有用户体验一致

### 7.2 成本覆盖方案

作为一个需要调用大模型API、运行服务器的个人项目，需要考虑基础运营成本：

| 成本项 | 预估月成本 | 说明 |
|--------|-----------|------|
| 服务器 | 100-500元 | 初期用户量小，可用轻量云服务器 |
| 大模型API | 200-2000元 | 视调用量而定，可通过缓存降低成本 |
| 域名/证书 | 约100元/年 | 基础开销 |
| 应用商店 | Apple开发者年费688元 | 如上架App Store |

**成本覆盖方式**（均为自愿，不影响任何功能使用）：
- **赞赏/打赏**：App内提供自愿赞赏入口，金额自定
- **爱发电/赞助**：在关于页面提供爱发电等赞助平台链接
- **API成本优化**：缓存高频解析、批量预生成、优先使用免费额度大的模型API
- 如果项目持续亏损，再考虑后续调整策略

### 7.3 用户增长策略（低成本/零成本）
- GitHub开源，吸引开发者社区关注
- 在考公相关论坛/社区（知乎、小红书、贴吧）分享产品体验
- 口碑传播，依靠产品体验获得用户自发推荐
- 用户反馈驱动迭代，建立忠实用户群

### 11.3 LLM统一适配层设计

```
通过配置切换不同LLM供应商：

llm:
  provider: deepseek    # 可切换为: qwen / zhipu / openai
  deepseek:
    api-url: https://api.deepseek.com/v1/chat/completions
    model: deepseek-chat
    api-key: ${DEEPSEEK_API_KEY}
  qwen:
    api-url: https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation
    model: qwen-turbo
    api-key: ${QWEN_API_KEY}
  zhipu:
    api-url: https://open.bigmodel.cn/api/paas/v4/chat/completions
    model: glm-4-flash
    api-key: ${ZHIPU_API_KEY}
```

- 统一接口屏蔽不同供应商的请求/响应格式差异
- 业务代码只需调用统一方法（如 `llmService.chat(prompt)`）
- 切换供应商只改配置，不改代码
- Python服务内部实现，Java服务通过HTTP调用

## 十二、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 题库内容版权问题 | 高 | AI辅助生成原创题目 + 收集整理公开真题 + 注意规避版权风险 |
| AI解析质量不稳定 | 高 | 用户反馈机制 + 持续优化prompt + 缓存优质解析 |
| 个人开发精力有限 | 高 | 聚焦MVP核心功能，严格控制范围；优先自动化，减少运维成本 |
| 大模型API成本 | 中 | 缓存高频解析 + 批量预生成 + 选择性价比高的国产模型 |
| 考试政策变化 | 中 | 及时跟踪政策动态，快速更新内容 |
| 服务器/运维压力 | 低 | 初期轻量部署，做好监控告警，随用户量弹性扩容 |
